# cbp-list

## Purpose
    the simple list renders unordered and ordered lists in accordance with the design system

## Functional Requirements

* the Simple List componenet API should support native UL/OL functionality and accessibility
* the Simple List maybe supplied slotted content for list items and nested lists 

## Technical Specifications

### User Interactions

* simple list is meant to display text in an itemized format,  no user interactions are detailed in the ux spec

### Responsiveness

* Lists are inline elements by default and should wrap, making them flexible at any screen size

### Accessibility

* [BETA] Simple Lists do not currently have any additional a11y support. to be added before full release

### Additional Notes and Considerations

* Simple list currently only support the unordered and ordered lists, more variants to be added in time
